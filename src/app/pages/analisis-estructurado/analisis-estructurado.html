Introducción

El análisis estructurado es una metodología utilizada para estudiar, comprender y representar el funcionamiento de una organización mediante la identificación de procesos, flujos de información, actores y almacenes de datos.

En el caso de Lácteos Pura, esta metodología permitió analizar los procesos relacionados con la producción de yogurt frutado natural, la gestión de inventarios, el control de ventas, la distribución y la administración de la información empresarial.

El resultado de este análisis constituye la base para el diseño del Sistema Integrado de Gestión Agroindustrial, permitiendo modelar el comportamiento del negocio mediante Diagramas de Flujo de Datos (DFD), Diccionario de Datos y Diagramas Entidad-Relación (DER).

Objetivo del Análisis

Analizar los procesos operativos y administrativos de Lácteos Pura mediante técnicas de análisis estructurado, identificando actores, procesos, entradas, salidas y flujos de información necesarios para el diseño de una solución informática que optimice la gestión de la producción, inventario, ventas y distribución.

C) Lista de Acontecimientos

Los acontecimientos representan los eventos externos que interactúan con el Sistema Integrado de Gestión Agroindustrial de Lácteos Pura y generan una respuesta por parte del sistema.

1. Reportes de producción

El administrador podrá solicitar reportes sobre la producción realizada en la empresa, permitiendo conocer la cantidad de yogurt elaborado, los lotes generados y el estado actual de la producción para apoyar la toma de decisiones.

2. Datos de materia prima

Los proveedores proporcionarán información sobre la entrega de leche, frutas e insumos necesarios para la elaboración de los productos. Estos datos serán registrados por el sistema para mantener actualizado el inventario de materias primas.

3. Registro de producción

El encargado de producción podrá registrar los datos correspondientes a cada proceso productivo, incluyendo fecha de elaboración, lote generado, cantidad producida y materias primas utilizadas.

4. Solicitud de productos

Los clientes podrán realizar pedidos de productos lácteos, permitiendo que el sistema registre la solicitud y verifique la disponibilidad existente en inventario antes de proceder con la venta o distribución.

5. Control de inventario

El encargado de almacén podrá registrar ingresos y salidas de productos, así como actualizar las cantidades disponibles cuando se realicen inventarios físicos o se detecten diferencias en el stock registrado.

6. Registro de ventas

El encargado de ventas podrá registrar las transacciones realizadas con clientes, permitiendo actualizar automáticamente el inventario y generar información para los reportes administrativos.

7. Datos de distribución

Los distribuidores podrán registrar la entrega de pedidos a los diferentes puntos de venta o clientes, permitiendo llevar un seguimiento de los productos despachados y entregados.

8. Reportes de control y trazabilidad

Las entidades reguladoras podrán solicitar información relacionada con la producción, lotes y fechas de vencimiento, permitiendo que el sistema genere reportes de control y trazabilidad de los productos elaborados por la empresa.

Diagrama Entidad-Relación

Modelo de Datos del Sistema

El Diagrama Entidad-Relación representa la estructura de la base de datos, mostrando las entidades principales del negocio, sus atributos y las relaciones entre ellas.

Entidades Principales

  • PRODUCTO: Artículos lácteos comercializados
  • CLIENTE: Compradores mayoristas y minoristas
  • PEDIDO: Solicitudes de compra
  • INVENTARIO: Control de stock
  • PRODUCCION: Registro de manufactura

Cardinalidades

  • Un cliente puede tener muchos pedidos
  • Un pedido tiene muchos detalles
  • Un producto tiene un registro de inventario
  • Un producto puede tener muchas producciones
